# EXOTIC_FINANCIAL_BUSINESS_PLAN

Date: 2026-03-29  
Scope: Freemium dating app economics for `exotic` (Russia-local strategy)

## 1) Benchmarks & Assumptions

- Freemium median conversion around D35: ~2.1% (reference benchmark used for target framing).
- Store commission hypothesis for net projections: **15%**.
- Model orientation: freemium-first (mass acquisition + conversion layers).

---

## 2) Core Monetization Diagnosis

Current value pillars sold:
- Visibility: `Boost`
- Direct initiative: `SuperLike`, first-message mechanics
- Social-risk control: `Rewind`, `IceBreaker`
- Status/power: `Premium badge`, `Platinum`, `ShadowGhost`, `Travel Pass`

Main corrections to enforce:
- Preserve `SuperLike` scarcity (do not over-distribute).
- Avoid short-pass cannibalization of monthly plans.
- Re-prioritize `IceBreaker` as a major conversion driver.
- Keep `online visibility` and `ShadowGhost` as high-status premium levers.

---

## 3) Mother Rule

Never allow low-cost plans to create unlimited power.

Target market tension:
- Free = controlled frustration
- Basic paid = comfort
- Mid paid = performance
- Premium paid = status + algorithmic advantage
- One-off purchases = impulse

## 8) Product Rules to Code

1. **One dominant promise per paywall**
- SuperLike = emotional impulse
- Boost = traffic performance
- IceBreaker = curiosity/anxiety relief
- Travel Pass = projection/expansion
- ShadowGhost = social power
- Premium badge = status

2. **No true unlimited for core market domination**
- Allowed unlimited: rewind, hide age/distance, chat translation
- Not allowed unlimited: boosts, superlikes, algorithmic exposure

3. **Pack value-price ratio discipline**
- Upper tier: +50% to +90% perceived value for +60% to +80% price

4. **Short passes are samples, not replacements**
- Day pass = curiosity-led, low domination
- Week pass = comfort trial, not monthly substitution

5. **Starter as first-purchase weapon**
- Recommended entry trigger offer:
  - 99 ₽
  - 5 SuperLikes
  - 2 Boosts
  - no ads for 3 days

6. **Event-triggered paywall logic**
- Multiple likes received → IceBreaker
- SuperLikes exhausted → SuperLike pack
- Low visibility in 24h → Boost
- Interesting match in another city → Travel Pass
- Message attempt before match → SuperLike
- Profile hesitation after pass → Rewind
- 3-day streak activity → Gold/Platinum
- Status-seeking behavior → Badge/Platinum

## 9) Algorithmic Monetization Rules

Feed scoring principle:

`Score = Compatibility×0.55 + Activity×0.15 + Recency×0.10 + ProfileQuality×0.10 + Monetization×0.10`

Constraint:
- Monetization weight should not exceed 10–15%.

| Tier | Feed impression bonus |
|---|---:|
| Free | 1.00 |
| Essential | 1.05 |
| Gold | 1.10 |
| Platinum | 1.18 |
| Active Boost | x1.8 to x2.5 (temporary) |

## 10) Scalable Economy Ratios

Target revenue split:

| Source | Target |
|---|---|
| Subscriptions | 50–60% |
| SuperLikes | 15–20% |
| Boosts | 15–20% |
| IceBreaker | 8–12% |
| Travel/ShadowGhost/etc. | 5–10% |

Guardrails:
- If subscriptions <45%: model is too volatile
- If token impulse <25% cumulative: in-app purchase loop is too weak

Anti-cannibalization limits:
- 7-day pass value <= 35–40% of monthly value
- Bundles must not beat Platinum in value per ruble
- High-status features remain outside entry tier
